# Как работает агент s01

В `s01` агент устроен максимально просто: есть история сообщений `messages`, один инструмент `bash` и цикл, который снова и снова спрашивает модель, нужен ли ей ещё один вызов инструмента.

---

## Общая схема

```text
User prompt -> LLM -> ответ
                    |
                    v
          stop_reason == "tool_use"?
             /                    \
           да                      нет
            |                       |
            v                       v
      выполнить bash          вернуть текст
      добавить tool_result
      в messages[]
            |
            v
        повторить цикл
```

---

## Шаг за шагом

### 1. Вход

Пользовательская задача добавляется в `messages` как сообщение `role: "user"`.

### 2. Вызов модели

```python
response = client.messages.create(
    model=MODEL,
    system=SYSTEM,
    messages=messages,
    tools=TOOLS,
    max_tokens=8000,
)
```

В API уходит:
- `system` — системная инструкция агента;
- `messages` — история диалога;
- `tools` — описание доступного инструмента `bash`.

### 3. Что возвращает API

Модель может вернуть:
- обычный текст;
- или блок `tool_use`, если она решила вызвать инструмент.

Типичный ответ выглядит так:

```python
response.content = [
    TextBlock(text="Сейчас выполню команду..."),
    ToolUseBlock(
        id="toolu_01abc...",
        name="bash",
        input={"command": "dir *.py"},
    ),
]
```

Ключевой момент: модель **не выполняет команду сама**. Она только формирует структурированный вызов инструмента.

### 4. Как `tool_use` превращается в запуск shell-команды

```python
for block in response.content:
    if block.type == "tool_use":
        output = run_bash(block.input["command"])
        results.append({
            "type": "tool_result",
            "tool_use_id": block.id,
            "content": output,
        })
```

Код:
- перебирает блоки в `response.content`;
- ищет блоки `tool_use`;
- берёт `block.input["command"]`;
- передаёт команду в `run_bash(...)`;
- возвращает результат в модель через `tool_result`.

---

## Инструмент `bash`

В `s01` доступен только один инструмент:

```python
TOOLS = [{
    "name": "bash",
    "description": "Run a shell command.",
    "input_schema": {
        "type": "object",
        "properties": {"command": {"type": "string"}},
        "required": ["command"],
    },
}]
```

Из этого описания модель понимает:
- как называется инструмент;
- что он делает;
- какие аргументы нужно передать.

Функция `run_bash` уже реально вызывает `subprocess.run(...)`, поэтому исполнение происходит локально в Python-коде, а не внутри API.

Ограничения:
- опасные команды блокируются;
- есть таймаут;
- длинный вывод обрезается.

## Внешний REPL

Снаружи есть простой REPL-цикл:
- читает ввод пользователя;
- добавляет его в историю;
- вызывает `agent_loop(history)`;
- печатает финальный ответ;
- повторяет, пока пользователь не завершит сессию.

История накапливается, поэтому агент помнит предыдущие шаги внутри одной сессии.

---

## Итог

`s01` показывает минимальное ядро агента:
- модель принимает решение;
- код исполняет инструмент;
- результат возвращается обратно в модель;
- цикл продолжается, пока `stop_reason == "tool_use"`.

Все следующие сессии усложняют инструменты, планирование и координацию, но базовый цикл остаётся тем же.
